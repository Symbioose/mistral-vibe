MeowMeowMeow

Tool builtin meow_meow_meow : le LLM écrit un script Python async qui orchestre des sub-agents en parallèle (agent(), parallel(), pipeline(), phase(), log()), avec validation statique stricte, journal de replay et TUI live. Chaque nœud ci-dessous porte le nom réel du composant dans le code.

1. Architecture d'ensemble

Le tool MeowMeowMeow valide le script, construit le MeowMeowMeowRuntime et relaie ses événements vers la TUI via ToolStreamEvent.data. Le runtime consulte le journal avant chaque spawn et délègue l'exécution réelle des agents à _AgentLoopSpawner (un AgentLoop par appel agent()).

2. Séquence d'un run nominal (+ resume)

Run nominal : validation, événement phases_planned (émis par le tool depuis meta.phases, pas par le runtime), fan-out de trois agents concurrents dont un avec retry de schéma, enregistrement au journal des seuls succès. Le bloc final montre le chemin resume : les appels agent() déjà réussis sont rejoués depuis journal.jsonl sans aucun spawn.

3. Cycle de vie d'un run

Un script rejeté ne crée jamais de run. Un run 0-agent est un échec explicite. La clé du replay : la re-invocation avec resume_from_run_id rejoue depuis journal.jsonl les appels agent() réussis (clé = hash SHA-256 de prompt + schema + agent_name + model) et ré-exécute uniquement le reste.

stateDiagram-v2
    state "Validation — parse_meow_meow_meow_script" as validation
    state "Rejeté — ToolError renvoyé au LLM" as rejete
    state "Running — MeowMeowMeowRuntime.run" as running
    state "completed" as completed
    state "failed" as failed
    state "cancelled" as cancelled

    [*] --> validation
    validation --> rejete : MeowMeowMeowScriptError, liste complète des violations
    rejete --> [*] : aucun run_id créé
    validation --> running : ParsedScript, run_id + script.py + journal créés
    running --> completed : main retourne et agents_spawned ≥ 1
    running --> failed : exception du script ou MeowMeowMeowError
    running --> failed : run 0-agent, échec explicite
    running --> cancelled : asyncio.CancelledError, pas de meow_meow_meow_finished émis
    completed --> [*] : MeowMeowMeowResult + result.json
    cancelled --> [*] : status CANCELLED renvoyé par le tool
    failed --> running : resume_from_run_id — replay des succès depuis journal.jsonl
    failed --> [*] : MeowMeowMeowResult failed + conseil de resume au LLM

4. Pipeline de validation du script

Ordre réel des vérifications dans parse_meow_meow_meow_script (script.py). Seule l'erreur de syntaxe est fail-fast ; toutes les autres violations sont accumulées puis rendues au LLM en une seule MeowMeowMeowScriptError listant la totalité des problèmes.
