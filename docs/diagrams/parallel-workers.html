Parallel workers (orchestrateur)

Le tool task étendu : profil worker write-capable, chaque worker isolé dans son propre worktree git, sémaphore max_parallel, commit et merge-back effectués par le parent, conflits reportés (jamais résolus automatiquement), UI « fat cat + kittens ». Chaque nœud porte le nom réel du composant dans le code. Note : l'isolation dérive du profil d'agent (AgentProfile.isolation) — il n'existe pas d'argument isolated sur TaskArgs.

1. Architecture d'ensemble

Le loop parent émet N tool calls task dans un même tour ; chaque invocation acquiert le sémaphore partagé (TaskToolState.semaphore, défaut max_parallel=4), résout le profil, prépare un worktree isolé, puis — le loop du worker terminé — commit et tente le merge-back côté parent. Les événements portent l'identité de l'agent et pilotent la UI fat cat / kittens.

2. Séquence : 3 workers dispatchés dans un tour

Trois tool calls task(agent=worker) dans un même tour de l'assistant : travail concurrent dans trois worktrees, puis pour chaque worker le parent commit sa branche et fait un dry-run de merge (git merge-tree). Le cas conflit est reporté au LLM, jamais résolu automatiquement.

        T-->>UI: SubagentStartedEvent : fat cat + kitten 1
        T->>W1: subagent_loop.act(task)
        W1-->>T: édite dans SON worktree (sandbox write limité au worktree)
    and worker 2
        T->>G: git worktree add -b vibe-worker-bbbbbbbb
        T-->>UI: SubagentStartedEvent : kitten 2
        T->>W2: subagent_loop.act(task)
        W2-->>T: édite dans SON worktree
    and worker 3
        T->>G: git worktree add -b vibe-worker-cccccccc
        T-->>UI: SubagentStartedEvent : kitten 3
        T->>W3: subagent_loop.act(task)
        W3-->>T: édite dans SON worktree
    end

    Note over T,G: chaque worker terminé : commit puis merge par le PARENT,
séquentiellement dans la coroutine de chaque invocation

    T->>G: commit_worktree branche 1 (git add -A, git commit)
    T->>G: merge_report : git merge-tree --write-tree (dry-run)
    alt propre et merge=auto
        T->>G: git merge --no-ff --no-edit vibe-worker-aaaaaaaa
        T-->>UI: SubagentFinishedEvent (merge_status=merged)
        T-->>LLM: TaskResult — suffixe [vibe-worker-aaaaaaaa: merged]
    else merge=manual (défaut)
        T-->>LLM: TaskResult — suffixe [branch vibe-worker-aaaaaaaa], branche laissée au parent
    end

    T->>G: commit_worktree branche 2
    T->>G: merge_report branche 2
    alt conflit détecté au dry-run
        T-->>UI: SubagentFinishedEvent (merge_status=conflicts)
        T-->>LLM: TaskResult — [vibe-worker-bbbbbbbb: conflicts] + conflicting_paths,
jamais résolu automatiquement
    end

    T->>G: commit_worktree branche 3 : rien à committer
    T-->>LLM: TaskResult — suffixe [no changes]

    Note over LLM,G: le parent peut ensuite merger manuellement les branches restantes
et relancer les tests sur le checkout (orchestration côté LLM, hors task.py)

3. Cycle de vie d'un worker

Chaque invocation task est une coroutine indépendante : un worker qui échoue produit un TaskResult failed sans emporter ses frères. Le commit est toujours effectué par le parent après la fin du loop du worker (le worker a pour consigne de ne jamais lancer git lui-même).

stateDiagram-v2
    state "Dispatché — tool call task, sémaphore acquis" as dispatched
    state "Worktree créé — branche vibe-agent-idcourt" as wtcreated
    state "Running — subagent_loop.act, écritures sandboxées au worktree" as running
    state "Committed — commit_worktree par le parent" as committed
    state "merged — git merge --no-ff dans le checkout parent" as merged
    state "conflicts — reporté au LLM, jamais auto-résolu" as conflicts
    state "not_attempted — merge=manual, branche laissée au parent" as notattempted
    state "no_changes — rien à committer" as nochanges
    state "failed — completed=false, frères non affectés" as failed

    [*] --> dispatched
    dispatched --> wtcreated : prepare_worktree_session, profil WORKER isolation WORKTREE
    wtcreated --> running : SubagentStartedEvent
    running --> committed : loop terminé, git add -A + commit par le parent
    running --> failed : exception du subagent, réponse annotée Subagent error
    committed --> merged : merge=auto et merge_report propre
    committed --> conflicts : merge_report sale, merge_status=conflicts
    committed --> notattempted : merge=manual, défaut
    running --> nochanges : aucun diff dans le worktree
    merged --> [*] : branche supprimée, worktree nettoyé
    nochanges --> [*] : branche supprimée
    conflicts --> [*] : branche conservée pour récupération manuelle
    notattempted --> [*] : branche conservée
    failed --> [*] : worktree conservé (keep_worktrees=on-failure)
